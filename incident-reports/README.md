# Incident Response Reports

## Overview

This section contains incident response reports generated from attack simulations performed within the Enterprise SOC Home Lab.

The reports follow a structured incident handling methodology based on industry-standard SOC and DFIR practices.

## Incident Response Lifecycle

1. Detection
2. Analysis
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

## Data Sources

* Wazuh SIEM
* Suricata IDS/IPS
* Zeek Network Security Monitoring
* Sysmon
* Velociraptor
* pfSense Firewall

## Objective

The purpose of these reports is to validate security controls, improve analyst readiness, and document incident handling procedures.

Each report includes:

* Executive Summary
* Detection Details
* MITRE ATT&CK Mapping
* Indicators of Compromise
* Containment Actions
* Lessons Learned
